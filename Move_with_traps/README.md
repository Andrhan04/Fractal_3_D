# Move_with_traps - Симуляция движения частиц с ловушками

Программа для запуска серии экспериментов по движению частиц в фрактальной области с учетом ловушек. Поддерживает многопоточное выполнение через OpenMP.

## 🎯 Назначение

Многократный запуск симуляций движения частиц с различными конфигурациями ловушек для статистического анализа:
- Времени жизни частиц в зависимости от количества и конфигурации ловушек
- Эффективности различных типов ловушек
- Пространственного распределения захваченных частиц

## 📋 Требования

- Компилятор C++17 с поддержкой OpenMP
- Зависимости из папки `Templates/`
- Конфигурационные файлы в `Sowing/Config/`
- Данные ловушек в `Sowing/Trap/`

## 🏗️ Архитектура

### Основные компоненты

1. **Чтение конфигураций** - `MyReader::read_config()`
2. **Инициализация частиц** - `input()` функция
3. **Основной цикл симуляции** - `programm()` функция
4. **Запись результатов** - `MyWriter` методы

### Ключевые особенности

- **Поддержка множества ловушек** - обработка массива конфигураций ловушек
- **Многопоточность** - параллельное выполнение повторных запусков
- **Эффективное управление памятью** - swap алгоритм для удаления "мертвых" частиц
- **Прогрессивное сохранение** - запись промежуточных результатов каждые 100,000 шагов

## 📊 Форматы данных

### Входной JSON
```json
[
    {
        "conf_id": 0,
        "exp_id": 0, 
        "repeat": 2,
        "steps": 10000000
    }
]
```

- **conf_id** - ID конфигурации фрактала/буфера
- **exp_id** - ID эксперимента для группировки результатов
- **repeat** - Количество повторных запусков
- **steps** - Количество шагов симуляции

### Конфигурация ловушек
В файле `Sowing/Config/Conf_{id}.json`:
```json
{
    "Bufer": 0,
    "Fractal": 0,
    "Size": 50,
    "Traps": [
        {
            "type": "spherical",
            "position": [10, 5, 15],
            "radius": 2.5
        }
    ]
}
```

## 🔧 Компиляция и запуск

### Компиляция
```bash
g++ -fopenmp -std=c++17 -I../Templates Move_with_traps.cpp -o Move_with_traps
```

### Запуск
1. Подготовьте входной файл `input.json`
2. Убедитесь в наличии конфигурационных файлов
3. Запустите программу:
```bash
./Move_with_traps
```

## 📈 Выходные данные

### Основные результаты
- **Experiments/exp_{try_id}.txt** - Позиции частиц на каждом шаге
- **Results/experiment_{exp_id}.json** - Статистика по экспериментам

### Статистические метрики
```json
{
    "conf_id": 0,
    "experiment_id": 0,
    "time_live": 15000,
    "Alive": 25,
    "Traps_count": 10,
    "particles_count": 100
}
```

- **time_live** - Время жизни половины частиц (N/2.7)
- **Alive** - Количество выживших частиц в конце
- **Traps_count** - Общее количество ловушек
- **particles_count** - Начальное количество частиц

## ⚙️ Алгоритм работы

### Основной цикл симуляции
1. **Инициализация** - Загрузка конфигурации и создание частиц
2. **Пошаговое движение** - Вызов `Particle::step()` для каждой активной частицы
3. **Проверка состояния** - Обнаружение "мертвых" частиц и их удаление через swap
4. **Сохранение прогресса** - Запись промежуточных результатов каждые 100k шагов
5. **Финализация** - Сохранение финальных результатов и статистики

### Управление ловушками
- Поддержка различных типов ловушек (сферические, цилиндрические)
- Конфигурирование через JSON
- Интеграция с фрактальной структурой

## 🎯 Особенности реализации

### Оптимизации
- **Эффективный swap** - Пользовательская функция обмена для класса Particle
- **Многопоточность** - Параллельное выполнение повторных запусков
- **Минимизация копирования** - Работа с ссылками и перемещением

### Обработка ошибок
- Проверка корректности входных данных
- Обработка исключений при чтении конфигураций
- Валидация позиций частиц после симуляции

## 🔗 Интеграция с другими компонентами

- `Draw_figure/Draw_point_positions.py` для отображения траекторий частиц

### Статистический анализ
- `Statistics/imperial_depends_from_traps.py` - Анализ зависимости времени жизни
- `Statistics/Heap_map.py` - Создание тепловых карт распределения
- `Statistics/Histogramm.py` - Построение гистограмм распределения

## ⚠️ Особенности работы

1. **Требуются ловушки** - Конфигурация должна содержать раздел "Traps"
2. **Многопоточность** - Количество потоков определяется системой (можно настроить через OMP)
3. **Память** - Требует значительной памяти для больших количеств частиц/шагов
4. **Время выполнения** - Длительные симуляции могут занимать часы/дни

## 📝 Примечания

- Для работы с ловушками необходимо предварительно сгенерировать конфигурации через `Start_trap`
- Программа автоматически обрабатывает все конфигурации ловушек в файле конфигурации
- Результаты группируются по experiment_id для удобства анализа
