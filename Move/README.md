## Move.cpp

Программа `Move.cpp` запускает серию экспериментов по движению частиц в фрактальной области.  
Конфигурации экспериментов задаются в виде JSON‑файлов, читаются с помощью `MyReader` и результаты
сохраняются через `MyWriter`.

### Основная логика

- Чтение списка запусков из входного JSON (метод `MyReader::StartMove`).
- Для каждого элемента списка:
  - берётся `conf_id` — идентификатор конфигурации фрактала и буферной зоны;
  - создаются объекты `Fractal` и `BuferZone`;
  - считываются начальные частицы (`Particle`) и проверяются их параметры;
  - выполняется `steps` шагов методом `Particle::step()` для каждой частицы;
  - результаты эксперимента сохраняются через `MyWriter`.

### Входной файл запусков

Пример файла [`input.exmple.json`](file:///c:/Users/user/source/Fractal_3_D/Move/input.exmple.json):

```json
[
    {
        "conf_id": 0,
        "exp_id": 0,
        "repeat": 2,
        "steps": 10000000
    },
    {
        "conf_id": "id конфигурации",
        "exp_id": "id эксперимента",
        "repeat": "сколько запусков в эксперименте",
        "steps": "сколько шагов сделают частицы"
    }
]
```

Поля:
- `conf_id` — ID конфигурации, по которому `MyReader` найдёт настройки фрактала и буферной зоны;
- `exp_id` — ID эксперимента, под которым будут сохранены результаты;
- `repeat` — сколько раз нужно запустить данную конфигурацию;
- `steps` — сколько шагов сделают частицы в каждом запуске.

### Особенности выполнения

- Программа использует OpenMP для параллельного выполнения запусков
- Отображается прогресс-бар для каждого запуска с процентом выполненных шагов
- Отображается общий прогресс выполнения всех запусков конфигурации

### Алгоритм движения частиц

**Основной цикл симуляции:**
1. Для каждой частицы в каждом запуске:
2. На каждом шаге вычисляется новое положение частицы
3. Проверяется столкновение с границами фрактала и буфера
4. При столкновении вычисляется отражение или поглощение
5. Статистика движения записывается для последующего анализа

**Особенности реализации:**
- Используется дискретное пошаговое моделирование
- Поддержка многопоточности через OpenMP для ускорения вычислений
- Эффективные алгоритмы проверки столкновений с цилиндрическими поверхностями
- Минимизация вычислений расстояний через пространственные оптимизации

### Формат входных данных

**Файл `input.json`:**
```json
[
  {
    "conf_id": 0,
    "exp_id": 0,
    "repeat": 5,
    "steps": 10000000
  },
  {
    "conf_id": 1,
    "exp_id": 1,
    "repeat": 3,
    "steps": 5000000
  }
]
```

**Параметры:**
- `conf_id` - ID конфигурации из Sowing/Config/
- `exp_id` - ID эксперимента для сохранения результатов
- `repeat` - количество повторений запуска (для статистики)
- `steps` - количество шагов моделирования на каждый запуск

### Формат выходных данных

**Файлы результатов:**
- `Experiments/exp_{id}.txt` - позиции частиц на каждом шаге
- `Results/experiment_{exp_id}.json` - метаданные и статистика эксперимента

**Содержимое файлов результатов:**
- Временные ряды координат частиц
- Информация о столкновениях и отражениях
- Статистика распределения частиц
- Параметры эксперимента и конфигурации

### Запуск программы

1. Подготовьте входной JSON с массивом конфигураций по образцу `input.exmple.json`.
2. Убедитесь, что конфигурации с нужными `conf_id` существуют в каталоге конфигураций,
   который использует `MyReader`.
3. Соберите исполняемый файл из `Move.cpp` и запустите его из каталога, ожидаемого `MyReader`,
   чтобы программа смогла найти входной JSON и конфигурации.

### Мониторинг выполнения

Программа отображает два уровня прогресса:

**Прогресс по запускам:**
```
Progress: [=========================                     ] 25/50 (50%)
```

**Прогресс по шагам внутри запуска:**
```
Step progress: [=======================>                 ] 67.8%
```

**Особенности мониторинга:**
- Реальное время выполнения оценки
- Автоматическая адаптация скорости отображения
- Поддержка прерывания с сохранением промежуточных результатов
- Логирование ошибок и предупреждений

### Оптимизации производительности
