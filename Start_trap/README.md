# Start_trap - Генератор конфигураций ловушек

Программа для создания и конфигурирования ловушек в фрактальной структуре. Генерирует различные конфигурации ловушек с заданными параметрами и интегрирует их в существующие конфигурации.

## 🎯 Назначение

Автоматическое создание наборов ловушек для экспериментов:
- Генерация ловушек со случайным размещением в сегментах фрактала
- Создание серий конфигураций с различным количеством ловушек
- Интеграция ловушек в существующие JSON конфигурации
- Обеспечение минимального расстояния между ловушками

## 📋 Требования

- Компилятор C++17
- Зависимости из папки `Templates/`
- Существующие конфигурации фракталов в `Sowing/Config/`
- Данные фракталов в `Sowing/Fractal/`

## 🏗️ Архитектура

### Основные компоненты

1. **Чтение конфигураций** - `MyReader::read_to_create_trap()`
2. **Генерация позиций** - `gen_posible_trap()` функция
3. **Проверка расстояний** - Валидация минимального расстояния между ловушками
4. **Сохранение результатов** - `MyWriter::write_traps()` и перезапись конфигураций

### Ключевые особенности

- **Интеллектуальное размещение** - Ловушки генерируются внутри сегментов фрактала
- **Контроль расстояний** - Гарантированное минимальное расстояние между ловушками
- **Серийная генерация** - Создание серий конфигураций с разным количеством ловушек
- **Интеграция с конфигурациями** - Автоматическое обновление JSON конфигураций

## 📊 Форматы данных

### Входной JSON
```json
{
    "N_min": 5,
    "N_max": 50,
    "Step": 5,
    "config_id": 0
}
```

- **N_min** - Минимальное количество ловушек
- **N_max** - Максимальное количество ловушек  
- **Step** - Шаг увеличения количества ловушек
- **config_id** - ID базовой конфигурации фрактала

### Выходные данные

#### Конфигурация ловушек
В файле `Sowing/Config/Conf_{id}.json` добавляется раздел:
```json
{
    "Traps": [101, 102, 103, 104, 105]
}
```

#### Файлы ловушек
Создаются в `Sowing/Trap/`:
- `Traps_101.txt` - Конфигурация с 5 ловушками
- `Traps_102.txt` - Конфигурация с 10 ловушками
- и т.д.

## 🔧 Компиляция и запуск

### Компиляция
```bash
g++ -std=c++17 -I../Templates Start_trap.cpp -o Start_trap
```

### Запуск
1. Подготовьте входной файл `input.json`
2. Убедитесь в наличии базовой конфигурации
3. Запустите программу:
```bash
./Start_trap
```

## 🎯 Алгоритм работы

### Генерация ловушек
1. **Загрузка фрактала** - Чтение структуры фрактала из конфигурации
2. **Подсчет сегментов** - Определение количества цилиндрических сегментов
3. **Генерация позиций** - Случайное размещение ловушек внутри сегментов:
   - Выбор случайного сегмента фрактала
   - Генерация точки внутри выбранного сегмента
   - Создание ловушки в полученной позиции

### Валидация размещения
4. **Проверка расстояний** - Гарантия минимального расстояния между ловушками
5. **Повторение генерации** - До достижения требуемого количества ловушек

### Сохранение результатов
6. **Сортировка ловушек** - Упорядочивание по идентификаторам
7. **Запись в файлы** - Сохранение конфигураций ловушек
8. **Обновление конфигурации** - Добавление ссылок на ловушки в JSON

## ⚠️ Особенности работы

1. **Зависимость от фрактала** - Ловушки генерируются внутри существующей структуры фрактала
2. **Минимальное расстояние** - Гарантируется расстояние ≥1 единица между ловушками
3. **Случайное размещение** - Позиции определяются случайным образом внутри сегментов
4. **Серийное создание** - Генерация нескольких конфигураций за один запуск

## 📝 Примечания

- Для работы требуется предварительно созданная конфигурация фрактала
- Ловушки размещаются только внутри рабочей области фрактала
- Программа создает последовательность конфигураций с разным количеством ловушек
- Идентификаторы ловушек назначаются автоматически последовательно

## 🔧 Настройка параметров

### Изменение минимального расстояния
Для изменения минимального расстояния между ловушками отредактируйте строку:
```cpp
if (traps[i].distance(posible_trap.position) <= 1) {
```
Замените значение `1` на желаемое минимальное расстояние.

### Изменение алгоритма генерации
Функция `gen_posible_trap()` отвечает за генерацию позиций ловушек. Можно модифицировать для:
- Различных распределений позиций
- Специфических паттернов размещения
- Привязки к определенным областям фрактала
