## Start_particle

Генерирует стартовые точки частиц в буферной зоне и формирует конфигурационный JSON для симуляции.

### Вход
- Файл `config.txt` в каталоге Start_particle c одной строкой:
  
  `BUFER_ID FRACTAL_ID N SIZE`
  
  где:
  - `BUFER_ID` — id буфера (Sowing/Bufer/Bufer_{id}.txt),
  - `FRACTAL_ID` — id фрактала (Sowing/Fractal/Pole_{id}.txt),
  - `N` — число генерируемых частиц,
  - `SIZE` — радиус цилиндров (для буфера и фрактала).

### Выход
- `Sowing/Particle/points_{id}.txt` — стартовые позиции частиц.
- `Sowing/Config/Conf_{id}.json` — конфигурация: связи Bufer/Fractal/Point/Size.

### Запуск
1. Откройте проект в Visual Studio и соберите `Start_particle`.
2. Поместите `config.txt` рядом с исполняемым файлом или запустите из каталога проекта.
3. Запустите исполняемый файл. В случае ошибок входа программа сообщит их в консоль.

### Что делает программа
- Читает геометрию буфера и генерирует `N` равномерно распределённых точек внутри наклонного цилиндра.
- Проверяет попадание каждой точки в буфер.
- Сохраняет точки и формирует `Conf_{id}.json` для дальнейшей симуляции.
