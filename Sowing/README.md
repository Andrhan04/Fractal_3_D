# Sowing - Данные для симуляции

Эта папка содержит входные данные для симуляции движения частиц в 3D фрактальных структурах.

## Структура каталога

### 📁 Config/
Содержит конфигурационные JSON файлы, которые связывают все компоненты симуляции:

**Формат файла `Conf_{id}.json`:**
```json
{
  "Bufer": <id_буфера>,
  "Fractal": <id_фрактала>,
  "Point": <id_частиц>,
  "Size": <радиус_цилиндров>
}
```

**Назначение полей:**
- `Bufer` - ID буферной зоны (соответствует файлу в Bufer/)
- `Fractal` - ID фрактальной структуры (соответствует файлу в Fractal/)
- `Point` - ID файла с частицами (соответствует файлу в Particle/)
- `Size` - радиус цилиндров для буфера и фрактала (в единицах координат)

### 📁 Bufer/
Содержит файлы с геометрией буферных цилиндров:

**Формат файла `Bufer_{id}.txt`:**
```
x1 y1 z1
x2 y2 z2
```

**Особенности:**
- Каждый файл содержит ровно две строки с координатами центров
- Буфер представляет собой наклонный цилиндр между этими точками
- Радиус цилиндра задается в конфигурационном файле
- Частицы генерируются равномерно внутри этого цилиндра

### 📁 Fractal/  
Содержит файлы с геометрией фрактальных структур:

**Формат файла `Pole_{id}.txt`:**
```
x1 y1 z1
x2 y2 z2
x3 y3 z3
...
```

**Особенности:**
- Каждая строка содержит координаты центра цилиндра
- Фрактал состоит из последовательности наклонных цилиндров
- Соседние цилиндры соединяются, образуя непрерывную структуру
- Радиус всех цилиндров одинаков и задается в конфигурации

### 📁 Particle/
Содержит стартовые позиции частиц:

**Формат файла `points_{id}.txt`:**
```
x1 y1 z1
x2 y2 z2
x3 y3 z3
...
```

**Особенности:**
- Каждая строка содержит координаты одной частицы
- Частицы генерируются внутри буферной зоны
- Распределение частиц - равномерное внутри цилиндра
- Количество частиц задается при генерации

## Процесс создания данных

1. **Подготовка геометрии**: Создайте файлы с центрами цилиндров в Bufer/ и Fractal/
2. **Генерация частиц**: Запустите Start_particle для создания точек в Particle/
3. **Создание конфигурации**: Программа автоматически создаст Conf_{id}.json в Config/
4. **Проверка**: Убедитесь, что все ID соответствуют существующим файлам

## Требования к данным

- Координаты должны быть в числовом формате
- Буфер должен содержать ровно 2 точки
- Фрактал должен содержать хотя бы 2 точки
- Все файлы должны использовать одинаковую систему координат
- Размеры должны быть согласованы (радиус не должен превышать минимальное расстояние между центрами)
